Lag En Todolist Med mongoDB

Krav

Må ha laget MongoDB, Mongoose lastet ned, basik Mongodb-compass, Express og Ejs Kunnskap

Data som skal lagres

Først lag en models mappe og lag en js fill eksemple Todo.js

Når man importer mogoose kan man bruke Mongoose elementer som .Schema og .model

.Schema er et document data structor og .model er konstruktører av et skjema og lager en forekomst av et dokument som tilsvarer poster i en database.

Express:

Først lag en routes mappe og lag en js fill eksemple index.js

først inport Router fra express og Todo fra todo.js i models mappen

.get for å få alle objektene i DB

.post er brukt til å lag en route hvor objecte blir post til siden

.patch er brukt itilfele det er en error som visser en feil medlig.

Bruke express med mogoose

Først lag en js fill dere du lagde de forige mapene eksemple app.js

først inport express fra express, mongoose fra mongoose og index fra index.js i routes mappen

mongoose.connect kobler coden til en mongoDB

.set setter et navn en verdi .set(name, value)

.use starter en req eller res function

.urlencoded er brukt for psot og put req

når .static er active så kan den ta array og object.

.listen er brukt til å feste tile spesifik port og host eks Listening on http://localhost:3000

Index.ejs:

Først lage en views mappe og lage en ejs fill eks index.ejs

lag en normale index me d en for loop for å hvis inholde til todolisten

bruke <% %> for å skrive js i Ejs på vær linje der det er js code

Er checkboxes checked:

lage en public mappe og en index.js i mappen

når en fille er i public mappen blir den set første og blir brukte av index.ejs når den er liket

code under er brukt for å lagre hvis boksen cheked eller ikke cheked

URLSearchParams er brukt til å formater eventer.cheked til riktig for søkmotoren

yaml fill:

først lagn .yaml fill og gi den et navn eks jav.yaml

systemlog brukes til å laggre alt errors og warnigs. destenaion velger hvor mongoDB snender all log output file eller syslog, hvis man velger file må man også ha path file er bedre å bruke en syslog fordi syslog logs medliger anderlerds som kan lage forvriene timestamps og path velger hvor dan skal lagres

storage er brukt til lagre data, journal er brukt så data er vaildert og recoverbel men det er bare active vis en dbPath er spcifyed

net er brukt for å speifisere en ip og port, bindIp er brukt for å speifisere ip

start MongoDB server:

mongod --config ./jav.yaml

nye terminal ok skrive npm start
